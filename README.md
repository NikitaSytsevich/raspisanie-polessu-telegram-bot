# Расписание ПолесГУ — Telegram-бот

Самостоятельный Telegram-бот для расписания спортивных объектов ПолесГУ. Это отдельный Vercel-проект: он не обращается к веб-приложению, не использует его API, Blob, cron или пользовательские смены.

## Что делает

- `/start` открывает единый экран расписания;
- все кнопки редактируют этот экран: листают дни, фильтруют объект, обновляют данные или открывают отдельный экран источников;
- в интерфейсе нет недельного экрана и команд-дублей: дату удобнее листать, а чат не заполняется сообщениями бота;
- загружает данные напрямую с официальных страниц ПолесГУ при запросе;
- утром первой проверкой обновляет карточку расписания на сегодня и присылает тихую сводку дня: одна строка на объект, без звука, вчерашняя сводка удаляется — в чате всегда ровно одна;
- с 8:00 до 18:00 по Минску каждые 10 минут сверяет официальные страницы и присылает отдельную сводку только при изменениях;
- хранит только технические `chat_id` и `message_id` карточки в Upstash Redis.

### Логика интерфейса

После `/start` бот показывает расписание на сегодня по всем объектам. Верхний ряд кнопок меняет дату, следующие два ряда — объект. Активный объект отмечен точкой. Название каждого объекта ведёт на его официальную страницу. «Обновить» принудительно перечитывает официальный сайт, а «Источники» открывает отдельный экран со всеми ссылками.

Бот хранит ID рабочей карточки. Повторный `/start` обновит её, а не создаст новую. Если карточку удалить вручную, следующий `/start` создаст одну новую.

Первая проверка QStash нового дня обновляет все зарегистрированные карточки. Telegram не присылает гарантированный push на редактирование старого сообщения, зато чат всегда остаётся с одной актуальной карточкой.

### Уведомления об изменениях

Upstash QStash вызывает `/api/check-changes` по одному cron-расписанию `*/10 5-14 * * *` (UTC): каждые 10 минут с 8:00 до 17:50 по Минску. Одно расписание занимает один слот из десяти на бесплатном плане, а суточный лимит в 1000 сообщений расходуется лишь на 6%. Бот сравнивает нормализованное расписание, а не HTML-разметку. Если появились или исчезли сеансы либо изменился статус объекта, основная карточка обновляется, а в чат приходит отдельная сводка. Первый запуск сохраняет исходное состояние и ничего не отправляет.

Если страница малого бассейна не содержит сеансов, бот сообщает, что расписание ещё не опубликовано. Это отличается от ошибки сети и не маскирует отсутствие данных.

Четыре страницы источника загружаются параллельно только по действию пользователя или одному утреннему запуску. В памяти тёплого инстанса есть короткий 90-секундный кэш, который не является хранилищем и не влияет на работу после cold start.

## Деплой

1. Создайте **отдельный** GitHub-репозиторий из этой папки и импортируйте его в Vercel как новый проект.
2. В Vercel Marketplace подключите **Upstash Redis** к этому проекту. Интеграция добавит `KV_REST_API_URL` и `KV_REST_API_TOKEN` автоматически; при прямом подключении Upstash используются `UPSTASH_REDIS_REST_URL` и `UPSTASH_REDIS_REST_TOKEN`.
3. В Vercel → Settings → Environment Variables добавьте остальные значения из `.env.example`:
   - `TELEGRAM_BOT_TOKEN` — токен от @BotFather;
   - `TELEGRAM_WEBHOOK_SECRET` — новая случайная строка из букв, цифр, `_` и `-`;
   - `APP_URL` — URL именно нового Vercel-проекта.
   - `CRON_SECRET` — отдельная случайная строка не короче 32 символов. Vercel передаст её cron-маршруту в заголовке авторизации.
   - `CHANGE_CHECK_SECRET` — отдельная случайная строка не короче 32 символов для QStash.
4. Сделайте production redeploy. Расписание cron из `vercel.json` будет создано только для production-деплоя.
5. Однократно откройте в браузере:

   `https://<ваш-домен>/api/telegram?setup=<TELEGRAM_WEBHOOK_SECRET>`

   В ответе должно быть `"ok": true`. Эта операция зарегистрирует новый webhook, команды и описание бота; старый webhook будет автоматически заменён.
6. Напишите боту `/start`. Это создаст и зарегистрирует единственную рабочую карточку для утреннего обновления.

Не передавайте ссылку из шага 3 другим людям и не сохраняйте её в истории публичных каналов: секрет в ней нужен только для начальной настройки. После успешной регистрации можно заменить его новой переменной и повторить setup.

## Локальная проверка

```bash
npm install
npm test
npx vercel dev
```

Публичный HTTPS URL нужен Telegram для webhook, поэтому локальный сервер сам по себе бот не подключит.

## Расписания проверок

Cron QStash задаётся в UTC; Минск — UTC+3 круглый год, поэтому `*/10 5-14 * * *` = каждые 10 минут с 8:00 до 17:50 по Минску. QStash передаёт `CHANGE_CHECK_SECRET` в заголовке авторизации, поэтому endpoint не доступен извне. Утреннее обновление основной карточки выполняется первой проверкой дня.
